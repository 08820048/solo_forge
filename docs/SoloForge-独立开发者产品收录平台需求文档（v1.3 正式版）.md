## 1. 项目概述

### 1.1 项目名称
**SoloForge**

### 1.2 项目副标题（Slogan）
- 中文：单人开发者锻造的产品宝库  
- 英文：The Forge for Solo Makers' Products

### 1.3 项目定位与目标用户
**SoloForge** 是一个面向**全球独立开发者**的产品收录、展示与变现平台，目标用户覆盖：
- **中国大陆、港澳台地区**的独立开发者（使用中文）
- **海外（欧美、东南亚等）**独立开发者（使用英文）
- **全球用户**：对独立产品感兴趣的用户、投资人、早期采用者、媒体

### 1.4 项目目标
- 成为国内外独立开发者首选的产品展示与发现平台
- 通过双语支持，降低语言壁垒，促进跨境交流与曝光
- **建立可持续的商业模式**，帮助优质产品获得更多曝光
- 积累全球优质单人开发产品，形成国际影响力

### 2.2 推荐算法设计

#### **核心原则**
1. **付费不等于作弊**：付费产品需保证质量（质量分 ≥ 60）
2. **算法透明化**：在帮助文档公开排序因子
3. **用户体验优先**：避免全屏付费产品，保持内容多样性

#### **排序权重公式**

```
总分 = 质量分(40%) + 热度分(30%) + 时效分(15%) + 付费加权(15%)
```

##### **1. 质量分（40%）**
| 指标 | 权重 | 说明 |
|------|------|------|
| 产品描述完整度 | 10% | 标题、Slogan、详细描述、标签齐全 |
| 媒体质量 | 5% | 至少 3 张截图 + Logo，视频加分 |
| 用户评分 | 10% | 5 星制，需 ≥5 个评价才生效 |
| 点击率（CTR） | 10% | 点击数 ÷ 展示数，7 天滚动 |
| 停留时长 | 5% | 详情页平均停留 >30s 加分 |

##### **2. 热度分（30%）**
| 指标 | 权重 | 说明 |
|------|------|------|
| 点赞数 | 10% | 最近 7 天新增点赞 |
| 收藏数 | 10% | 最近 7 天新增收藏 |
| 外部访问量 | 5% | 从产品官网点击「查看收录」的回流量 |
| 社交分享 | 5% | Twitter、LinkedIn 分享次数 |

##### **3. 时效分（15%）**
- **新产品加成**：上线 30 天内额外 +20 分
- **更新活跃度**：
  - 7 天内更新：+10 分
  - 30 天内更新：+5 分
  - 超过 90 天未更新：-10 分

##### **4. 付费加权（15%）**
- **Featured**：总分 × 1.5（但质量分 <60 不上首页）
- **Sponsored**：总分 × 1.3
- **Free**：无加权

#### **特殊规则**
1. **冷启动保护**：新产品上线前 7 天保证至少 100 次首页展示
2. **多样性惩罚**：同一开发者的产品在同一页面不连续出现
3. **黑名单机制**：
   - 用户举报 ≥5 次 → 人工审核
   - 质量分 <40 → 自动下架
   - 恶意刷量 → 永久封禁

## 3. 国际化（i18n）需求

### 3.1 支持语言
- **首要语言**：
  - 英文（en-US）：默认语言，面向国际用户
  - 中文简体（zh-CN）：面向中国大陆及华语用户
- **后续扩展**（二期）：中文繁体（zh-TW）、日语（ja）、韩语（ko）

### 3.2 语言切换功能
- **全局切换器**：位于导航栏右上角（地球图标 🌐 + 下拉菜单）
- **自动检测**：根据浏览器 `Accept-Language` 自动选择语言
- **持久化**：使用 Cookie 或 localStorage 记住用户偏好
- **URL 策略**：支持子路径模式（如 `/en/`、`/zh/`），便于 SEO

### 3.3 内容翻译策略

| 内容类型 | 翻译策略 |
|---------|---------|
| UI 文本（按钮、导航、提示） | 全部静态翻译，使用 i18n 词条 |
| 系统生成内容（错误提示、通知） | 全部翻译 |
| 用户生成内容（产品名称、描述） | 不强制翻译，用户提交时可选语言 |
| 产品分类/标签 | 提供双语映射（如 "AI Tools" ↔ "AI 工具"） |
| SEO 元信息 | 每种语言生成独立版本 |

### 3.4 技术实现方案

**前端（Next.js）**：
- 使用 **next-intl**（App Router 原生支持）
- 翻译文件结构：
  ```
  messages/
  ├── en.json
  ├── zh.json
  ```
- 动态路由：`app/[lang]/page.tsx`

**后端（Actix Web）**：
- API 根据 `Accept-Language` 返回多语言错误信息
- 响应头添加 `Content-Language`

**数据库（Supabase）**：
- 用户生成内容原样存储，不做翻译处理

## 4. 用户认证系统

### 4.1 登录方式
- **Google OAuth**（通过 Supabase Auth）
- **GitHub OAuth**（通过 Supabase Auth）
- 不支持邮箱密码登录（降低开发成本）

### 4.2 用户权限分级

| 角色 | 权限 |
|------|------|
| **游客** | 浏览产品、搜索、筛选 |
| **注册用户** | 点赞、收藏、评论、提交产品 |
| **开发者** | 管理自己提交的产品、查看数据面板 |
| **Pro 会员** | 享受会员特权（见 2.1 模式二） |
| **管理员** | 审核产品、管理用户、配置 Featured 位 |

### 4.3 产品提交流程
1. 用户登录后点击「提交产品」
2. 填写表单：
   - 基础信息：产品图标、产品名称、Slogan、详细描述、官网链接
   - 分类标签：选择 1-3 个分类
   - 语言选择：英文 / 中文 / 双语
3. 提交后进入**待审核**状态
4. 管理员审核通过后，产品上线展示

### 6.3 关键 API 端点

未登录用户只能浏览内容，不能进行任何产品相关的操作，比如后续的产品提交、点赞、收藏等，这些需要登录之后才能操作。



**产品相关**：
- `GET /api/products` - 获取产品列表（支持分页、筛选、排序）
- `GET /api/products/:slug` - 获取产品详情
- `POST /api/products` - 提交新产品（需登录）
- `PUT /api/products/:id` - 更新产品信息（仅限作者）
- `POST /api/products/:id/upgrade` - 升级产品到 Sponsored/Featured


**用户相关**：
- `POST /api/auth/login` - OAuth 登录（Supabase 处理）
- `GET /api/users/me` - 获取当前用户信息
- `GET /api/users/me/products` - 获取我提交的产品



**统计相关**：
- `POST /api/analytics/track` - 埋点上报（前端调用）
- `GET /api/products/:id/stats` - 获取产品数据（仅限作者）
